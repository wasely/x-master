# X Master

X Master is a database-backed writing workspace for X. It learns from the posts you save into your own Supabase vector database, then uses those examples to help you write new posts, improve drafts, and generate replies that stay close to your saved voice.

The repo includes:

- A Next.js app for generating posts, managing drafts, and curating the style library
- A Chrome extension for saving tweets from X, generating replies inline, polishing reply drafts, and sending X videos to Google Drive
- A Supabase pgvector database for storing positive examples and rejected patterns, with Chroma-compatible app helpers

## What The App Does

### Style library

Save tweets you like directly from X into the hosted vector database. Those examples become the reference set for future generations.

### Draft and post generation

Generate new X posts from a topic or from TikTok source material while matching the structure and tone of the examples in your library.

### Database-backed reply generation

Every inline reply generated by the extension is grounded in:

- Positive examples from your saved database
- Rejected patterns from your rejection collection
- The target tweet itself

The reply flow is designed to stay short, accurate, and stylistically consistent instead of rewriting into a random generic voice.

### Reply enhancer in the extension popup

Paste a rough reply draft into the popup and lightly polish grammar, clarity, and flow without changing the wording more than necessary.

### X video to Google Drive

If Google Drive is configured, the extension adds a Drive button on X videos so you can save them directly to a selected folder.

## Quick Start

### 1. Install dependencies

```bash
npm install
```

### 2. Create your local env file

Copy `.env.example` to `.env.local` and fill in the values you need.

Minimum setup:

```env
OPENROUTER_API_KEY=your_openrouter_api_key
OPENROUTER_MODEL=google/gemini-2.5-flash-lite
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
HUGGINGFACE_API_TOKEN=hf_your_token_here
```

Optional vector settings:

```env
VECTOR_COLLECTION=tweet_examples
VECTOR_REJECTION_COLLECTION=tweet_rejections
HUGGINGFACE_EMBEDDING_MODEL=BAAI/bge-small-en-v1.5
```

### 3. Set up the vector tables

Run `supabase/schema.sql` in the Supabase SQL editor. It creates:

- `tweet_examples`
- `tweet_rejections`
- `match_tweet_examples`
- `match_tweet_rejections`

### 4. Seed saved examples

This repo includes `data/chroma-seed.json`, exported from the local Chroma store. Push it into Supabase with:

```bash
npm run db:seed
```

### 5. Start the app

```bash
npm run dev
```

### 6. Open the app

Go to:

```text
http://localhost:3000
```

If port `3000` is busy, the dev script will move to the next available port and print it in the terminal.

## Chrome Extension Setup

1. Open `chrome://extensions`
2. Enable `Developer mode`
3. Click `Load unpacked`
4. Select the `chrome-extension/` folder
5. Open the extension popup and confirm the detected app URL

The extension defaults to the hosted Vercel app and also supports `localhost:3000` through `localhost:3003`. If the Vercel deployment has Deployment Protection enabled, add a Vercel automation bypass token in the extension popup.

## Extension Features

### Save tweets to the style library

Right-click any tweet on X and choose:

- `Add tweet to style library`

### Save tweets as drafts

Right-click any tweet on X and choose:

- `Save tweet as draft`

This stores the draft in the app and also attempts to place it into X's draft flow.

### Generate replies inline on X

On each tweet, X Master adds its own button near the top-right controls. Clicking it lets you:

- Choose `Length`
- Choose `Feeling`
- Generate a reply from your saved database
- Insert the generated text directly into the reply box

### Enhance a reply from the popup

Use the popup if you already wrote a draft and only want a light cleanup instead of a full rewrite.

### Save X videos to Google Drive

When a tweet contains a video, the extension overlays a `Drive` button on the player.

## Google Drive Setup

Drive upload is optional.

### My Drive folder

Use OAuth credentials:

- `GOOGLE_DRIVE_FOLDER_ID`
- `GOOGLE_DRIVE_CLIENT_ID`
- `GOOGLE_DRIVE_CLIENT_SECRET`
- `GOOGLE_DRIVE_REFRESH_TOKEN`

To generate a refresh token after adding the client ID and client secret:

```bash
npm run drive:auth
```

### Shared Drive folder

You can use a service account instead. Share the target Shared Drive with the service-account email, then configure one of:

- `GOOGLE_DRIVE_SERVICE_ACCOUNT_EMAIL` and `GOOGLE_DRIVE_PRIVATE_KEY`
- `GOOGLE_DRIVE_SERVICE_ACCOUNT_JSON`
- `GOOGLE_DRIVE_SERVICE_ACCOUNT_JSON_PATH`

Important:

- Service accounts cannot upload into a normal personal My Drive folder they do not own
- Shared Drive targets are the safer path for service-account uploads

## Privacy And Local Data

This project is set up to avoid shipping personal local data by default:

- `.env.local` is ignored
- `.runtime/` is ignored
- `.public-runtime/` is ignored
- local screenshots and logs are ignored
- `data/chroma-seed.json` is intentionally tracked so the hosted database can be recreated from the saved examples

Before publishing your own fork, keep secrets out of tracked files and avoid committing local runtime data.

## Notes

- Reply generation will fail until you add examples to the style library
- The extension is designed for X's current DOM and may need small selector updates if X changes its UI
- If you use a different OpenRouter model, keep one that is fast enough for inline extension actions
